# api

NestJS 11 REST + tRPC server. Runs on port 8000.

## Scripts

```bash
bun run dev              # start with file watching (bun runs TS directly)
bun run typecheck        # tsc --noEmit
bun run generate         # generate tRPC types once
bun run generate:watch   # regenerate tRPC types on every router change
bun run test             # unit tests (Jest)
bun run test:e2e         # e2e tests
bun run build            # compile to dist/ for production (nest build)
```

## Folder Structure

Shared infrastructure lives under `src/shared/`:

```
src/
├── shared/
│   └── trpc/
│       ├── trpc.ts          # initTRPC instance
│       └── app.router.ts    # tRPC routers (add new ones here)
├── @generated/
│   └── server.ts            # DO NOT EDIT — generated by nestjs-trpc
├── app.module.ts
└── main.ts
```

All future shared concerns (DB, constants, types, utils) go under `src/shared/<concern>/`.

## tRPC — Adding a New Router

1. Create `src/shared/trpc/<feature>.router.ts`:

```ts
import { Injectable } from "@nestjs/common";
import { Input, Query, Router } from "nestjs-trpc";
import { z } from "zod";

@Injectable()
@Router()
export class FeatureRouter {
  @Query({
    input: z.object({ ... }),
    output: z.object({ ... }),
  })
  myProcedure(@Input() input: { ... }) {
    return { ... };
  }
}
```

2. Add to `providers: []` in `app.module.ts`
3. Run `bun run generate`

The class name becomes the sub-namespace on the client: `FeatureRouter` → `trpc.featureRouter.*`.

## tRPC — Mutations

Use `@Mutation()` instead of `@Query()` for write operations (same signature).

## ⚠️ After changing any procedure signature

Run `bun run generate` (or keep `generate:watch` running) before typechecking `web-ui`.
